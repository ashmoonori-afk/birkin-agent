# Documentation

## When to Use

Use this skill for README, architecture notes, runbooks, summaries, and review artifacts.

## Procedure

1. State scope and date for practical docs.
2. Separate evidence from interpretation.
3. Include commands that were actually run or are ready to run.
4. Keep durable deliverables in English unless instructed otherwise.

## Pitfalls

- Do not include unsupported claims.
- Do not write marketing copy for operator tooling.

## Verification

- A user can follow the document without hidden context.
